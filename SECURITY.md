# Security Policy

Bear-Arms is security-sensitive software. Treat false positives, false negatives, and destructive side effects as product risks.

## Reporting

Report vulnerabilities privately. Include:

- affected command or feature
- host platform
- reproduction steps
- expected versus actual behavior
- whether quarantine or audit behavior is affected

## Baseline Expectations

- Scanning should be safe by default.
- Quarantine actions must remain explicit and auditable.
- Findings should be structured and reproducible.
- Secrets, sample payloads, and customer data must never be committed.
